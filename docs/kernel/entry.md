この章では、カーネルのエントリーポイントについて記述します。
実装されているファイルは[entry.c](../../src/kernel/entry.c)です。

## 概要
カーネルのエントリーポイントを定義するファイルです。ブートローダーからカーネルに制御が移ると、最初にこの`kernel_entry`関数が呼ばれます。この関数はブート情報を受け取り、メイン関数`kmain`に制御を渡します。

## 関数 / API

#### `void kernel_entry(BOOT_INFO *boot_info)`
カーネルのエントリーポイントです。`.text.kernel_entry`セクションに配置され、ブートローダーから最初に呼ばれる関数です。ブート情報を`kmain`に渡します。

引数:
  - boot_info(BOOT_INFO*): ブートローダーから渡されるブート情報

## 定数 / 定義

このファイルには定数定義はありません。

## 構造体

#### `BOOT_INFO`
ブートローダーから渡されるブート情報を格納する構造体です（boot_info.hで定義）。

## 実装の詳細

### セクション配置
`__attribute__((section(".text.kernel_entry")))`により、この関数は特定のセクション`.text.kernel_entry`に配置されます。これはリンカスクリプトでカーネルの先頭に配置されるため、ブートローダーから確実に呼び出されます。

### 制御フロー
1. ブートローダーが`kernel_entry`を呼び出す
2. `kernel_entry`が`kmain`を呼び出す
3. `kmain`がカーネルの初期化とメインループを実行
